# humanizer-tw：中國用語滲入問題與解決方案

## 問題描述

humanizer-tw skill 的核心功能是將 AI 生成的中文文字「口語化」，使其更像台灣人寫的。然而在口語化過程中，LLM 會產出中國用語（如「幹活」「拉架」「靠譜」），而非台灣人日常說的話。

### 具體症狀

輸入：「我們需要協同工作，別急著消除摩擦。」

- **期望產出**：「我們得一起做事，別急著勸架。」
- **實際產出**：「咱們一起幹活，別急著拉架。」（混入中國口語）

問題不僅限於口語層面，書面／科技用語也會滲入：

- 「資訊」被寫成「信息」
- 「影片」被寫成「視頻」
- 「預設」被寫成「默認」
- 「伺服器」被寫成「服務器」

### 影響範圍

- 所有經 humanizer-tw 處理的文字都可能混入中國用語
- 口語化程度越高，滲入風險越大（因為 LLM 的「口語」概念偏向訓練資料中佔比最高的中國中文）
- 使用者可能無法察覺某些詞彙的差異（如「小夥伴」「給力」已部分進入台灣網路用語，但在正式寫作中仍不適合）

---

## 根本原因分析

### 1. LLM 訓練資料的中國中文佔比偏高

主流 LLM（GPT、Claude、Gemini 等）的中文訓練資料中，中國簡體中文的比例遠高於台灣繁體中文。這導致模型的「中文口語」內部表徵偏向中國口語。

當 skill 指示「口語化」時，模型傾向從佔比最高的中國口語中取詞，而非台灣口語。

### 2. Skill 缺乏明確的防護機制

原版 humanizer-tw 涵蓋了 8 大類 AI 寫作問題：

1. 開場白與連接詞
2. 互聯網黑話
3. 翻譯腔
4. 書面語過重
5. 公式化結構
6. 結尾套話
7. 語氣問題
8. 節奏問題

但沒有任何一個類別明確處理「中國用語 vs 台灣用語」的差異。Skill 只說「使文字更像台灣人寫的」，卻沒有提供具體的對照表讓模型知道哪些詞該避開、該用什麼替代。

### 3. 「口語化」指令的歧義

「口語化」對 LLM 而言是模糊指令。模型可能理解為：
- 把書面語換成口語 ✓
- 用中文世界裡最常見的口語表達 ✗（這會導向中國口語）

沒有明確約束時，模型會選擇訓練資料中出現頻率最高的口語表達。

### 4. 書面用語差異被忽略

中國與台灣的差異不僅在口語，書面用語（尤其科技領域）也有系統性差異。這些差異不是「口語化」能解決的，需要獨立處理。

---

## 解決方案

### 設計思路

核心策略：**用具體的對照表取代模糊的指令**。

LLM 對「避免中國用語」這類抽象指令的遵循度不穩定，但對「把 X 換成 Y」的具體對照表遵循度很高。因此解決方案以建立完整的對照表為核心。

### 實施內容

#### 1. 新增「類別九：中國用語滲入」

在 SKILL.md 的類別八之後新增完整類別，包含：

- 問題說明：口語化 ≠ 中國用語，LLM 訓練資料偏差的背景
- 口語對照表（20 組）
- 書面／科技對照表（22 組）
- 改寫範例（6 組，含正確與錯誤示範）

#### 2. 核心規則速查新增第 6 條

```
6. **避免中國用語** - 口語化 ≠ 中國用語，用台灣人日常說的話
```

從 5 條擴充為 6 條，確保這條規則在速查時被看見。

#### 3. 快速檢查清單新增一項

```
- ✓ **有中國用語（幹活、靠譜、視頻、默認）？** 換成台灣說法
```

範例同時涵蓋口語（幹活、靠譜）和書面（視頻、默認），提醒兩個層面都要檢查。

#### 4. references/phrases.md 新增對照段落

在「書面語過重」和「快速對照」之間新增完整的「中國用語 → 台灣說法」段落，分為口語和書面／科技兩個子表，方便快速查閱。

## 術語選擇：「中國用語」而非「大陸口語」

在開發過程中，曾考慮三種術語：

| 術語 | 優點 | 缺點 |
|------|------|------|
| 大陸口語 | 語意精確指口語層面 | 無法涵蓋書面差異；「口語」限縮範圍 |
| 中國大陸用語 | 正式、完整 | 5 字太長；偏官方語感，與 skill 口語調性不一致 |
| **中國用語** | 4 字精簡；涵蓋口語+書面；台灣日常用法 | 無 |

最終選擇「中國用語」，理由：
1. **涵蓋範圍正確**：同時包含口語（幹活、靠譜）和書面（信息、視頻）差異
2. **符合台灣日常用法**：台灣人說「中國用語」，不太會特別加「大陸」
3. **精簡**：skill 本身在教人寫得精簡，用詞以身作則
4. **語境無歧義**：在繁體中文台灣語境下，「中國」指的就是中華人民共和國

## 關鍵洞察

### 對照表比指令重要

測試發現，LLM 是否避開中國用語，**主要取決於對照表中的具體詞對**，而非類別標題或說明文字的措辭。

也就是說：

- 類別標題叫「大陸口語滲入」或「中國用語滲入」，對產出影響極小
- 真正起作用的是那 42 組「X → Y」的對照表
- 對照表越具體、越完整，防護效果越好

這個洞察的啟示是：**未來擴充防護範圍，應優先擴充對照表，而非修改說明文字。**

### Skill 系統可能有快取

全域安裝的 skill 修改原始碼後，skill 系統可能載入舊版內容。解決方式是重新執行 `npx skills install --all -g`。

## 後續可擴充方向

1. **擴充對照表**：持續收集使用者回報的漏網中國用語，補入對照表
2. **分級處理**：某些中國用語已進入台灣網路用語（如「給力」），可考慮標記為「視語境決定」而非一律替換
3. **自動檢測**：開發一個前處理步驟，自動掃描文字中的中國用語並標記，再交由 skill 替換
4. **測試集**：建立一組包含各種中國用語的測試文字，每次修改 skill 後跑測試驗證效果
