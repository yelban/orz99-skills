# AI 寫作反模式

所有句型、結構層級的 AI 寫作問題集中於此。SKILL.md 引用本檔作為唯一模式來源。

---

## 開場白套話

直接刪除，從內容開始。

### 時代開場
- 「隨著...的發展」「隨著...的興起」「隨著...的普及」
- 「在...的背景下」「在...的浪潮中」
- 「當今時代」「在這個...的時代」「當下」
- 「進入...時代」「踏入...的新紀元」

### 共識開場
- 「眾所周知」「不言而喻」「顯而易見」
- 「毋庸置疑」「不可否認」「毫無疑問」
- 「正如大家所知」「想必大家都知道」

### 宣告式開場
- 「值得一提的是」「值得注意的是」「值得關注的是」
- 「不得不說」「必須承認」「坦白說」
- 「有趣的是」「有意思的是」「令人驚訝的是」

---

## 連接詞濫用

AI 文章過度依賴這些詞來串連段落，大量刪減。

### 遞進連接
- 「此外」「另外」「不僅如此」「除此之外」
- 「更重要的是」「更為關鍵的是」「更值得關注的是」
- 「與此同時」「同時」「另一方面」

### 列舉連接
- 「首先...其次...再次...最後...」
- 「第一...第二...第三...」
- 「一方面...另一方面...」

### 總結連接
- 「總的來說」「總而言之」「綜上所述」
- 「歸根結底」「說到底」「從本質上來說」

---

## 結尾套話

直接刪除或用具體結論替代。

### 展望類
- 「讓我們拭目以待」「未來可期」「前景可期」
- 「相信在...的努力下」「期待...的到來」
- 「讓我們共同期待」「讓我們一起見證」

### 口號類
- 「為...貢獻力量」「為...添磚加瓦」
- 「讓我們攜手共進」「讓我們並肩前行」
- 「開啟...的新篇章」「書寫...的新故事」

### 反思類
- 「這是一個值得思考的問題」「這值得我們深思」
- 「或許，答案就在...」「也許，這就是...的意義」
- 「這需要時間來驗證」「時間會給出答案」

### 許可類
- 「這沒關係」「這是正常的」「這很常見」
- 「不要擔心」「不必焦慮」

### 客套類
- 「非常感謝您的閱讀」→ 刪除
- 「希望這篇文章對您有所幫助」→ 刪除
- 「如有不足之處，請多多指教」→ 刪除
- 「歡迎在評論區留言」→ 刪除或自然帶過

## 翻譯腔結構

### 「這是一個...的事情」結構

英文 "It is a... thing" 的直譯，刪除框架直接說。

| 翻譯腔 | 自然中文 |
|--------|----------|
| 這是一個很重要的事情 | 這很重要 |
| 這是一個值得探討的問題 | 值得探討 |
| 這是一個令人興奮的消息 | 令人興奮 |
| 這是一個不容忽視的現象 | 不能忽視 |

### 過長的定語從句

英文 relative clause 的影響，在名詞前堆疊過多修飾語。

| 翻譯腔 | 自然中文 |
|--------|----------|
| 在這個充滿機遇與挑戰並存的時代裡的年輕人 | 這個時代的年輕人面對機遇和挑戰 |
| 一個能夠幫助用戶快速完成任務的工具 | 一個工具，能幫用戶快速完成任務 |
| 那些在過去幾年中取得顯著成就的公司 | 過去幾年表現好的公司 |

### 被動語態過度使用

| 翻譯腔 | 自然中文 |
|--------|----------|
| 這個問題被認為是 | 大家認為這個問題是 |
| 這項政策被廣泛討論 | 大家都在討論這項政策 |
| 報告被提交給管理層 | 我們把報告交給管理層 |
| 這個方案被採納了 | 我們採納了這個方案 |

## 公式化段落結構

### 開頭-中間-結尾公式

AI 喜歡：開頭總結 + 中間三點展開 + 結尾金句。打破它。

### 三段式論證

每個觀點都用「論點→論據→結論」結構，讀起來像考試作文。

### 否定式排比

「不是 X，而是 Y」「不僅 X，更是 Y」的機械堆疊。

| 問題模式 | 處理方式 |
|----------|----------|
| 「不是 A，而是 B」 | 直接說 B |
| 「不僅 A，更是 B」 | 說「A，也是 B」或只說 B |
| 「不單單是 A，還是 B，更是 C」 | 挑最重要的一個說 |

## 中式 AI 句型

LLM 生成的中文常出現以下句型結構，這些是詞彙替換無法解決的「中式 AI 語法」：

| AI 句型 | 自然中文 |
|---------|----------|
| 進行了一個 X 的優化 | 優化了 X |
| 起到了 Y 的作用 | 幫助了 Y / Y 的功勞 |
| 給到了很好的反饋 | 給了好的回饋 |
| 對 X 進行了深入的分析 | 深入分析了 X |
| 在 X 方面取得了顯著的成效 | X 成效顯著 |
| 基於 X 的前提下進行 Y | 在 X 之下做 Y |
| 圍繞 X 展開了 Y 的工作 | 針對 X 做了 Y |
| 通過 X 的方式來實現 Y | 用 X 達成 Y |

**共同特徵：** 名詞化動詞 + 冗餘介詞 + 多餘框架。直接用動詞表達。

## 語氣問題

### 缺乏個人觀點

AI 傾向使用「有人認為」「專家指出」等規避主觀立場。

| 規避 | 直接 |
|------|------|
| 有人認為這很重要 | 這很重要 |
| 專家指出這是趨勢 | 這是趨勢 |
| 研究表明效果顯著 | 效果顯著（或引用具體研究） |
| 人們普遍認為 | 我覺得 / 大家覺得 |

### 過渡詞依賴

AI 不信任讀者能跟上思路，每個轉折都要加連接詞。刪掉它們，讓句子自己說話。

### 同義詞機械替換

AI 為避免重複，刻意替換同義詞，反而不自然。

> 「這位工程師→這名技術人員→該名從業者→這位專業人士」

**原則：** 重複沒關係。用同一個詞比硬換更自然。

## 個性與靈魂

> **注意：** 此章節僅適用於非正式文體（部落格、社群貼文、散文）。正式公文、技術文件、法律合約、財報等文體請跳過本章節。

避免 AI 模式只是工作的一半。無菌、沒有聲音的寫作和機器生成的內容一樣明顯。

### 缺乏靈魂的寫作跡象（即使技術上「乾淨」）：
- 每個句子長度和結構都相同
- 沒有觀點，只有中立報道
- 不承認不確定性或複雜感受
- 適當時不使用第一人稱視角
- 沒有幽默、沒有鋒芒、沒有個性
- 讀起來像維基百科文章或新聞稿

### 如何增加語調：

**有觀點。** 不要只是報告事實——對它們做出反應。

**變化節奏。** 短促有力的句子。然後是需要時間慢慢展開的長句。混合使用。

**承認複雜性。** 真實的人有複雜的感受。「這令人印象深刻但也有點不安」勝過「這令人印象深刻」。

**適當使用「我」。** 第一人稱不是不專業——而是誠實。

**允許一些混亂。** 完美的結構感覺像演算法。跑題和半成型的想法是人性的體現。

**對感受要具體。** 不是「這令人擔憂」，而是具體描述擔憂的場景。

## 快速對照

| 問題類型 | 徵兆 | 處理 |
|----------|------|------|
| 時代開場 | 「隨著 AI 的發展」 | 刪除，直接開始 |
| 共識開場 | 「眾所周知」 | 刪除 |
| 連接詞 | 「此外」「不僅如此」 | 刪除或減少 |
| 結尾套話 | 「讓我們拭目以待」 | 刪除或用具體結論 |
| 翻譯腔 | 「這是一個...的」 | 刪除框架，直接說 |
| 定語過長 | 名詞前超過 10 字修飾 | 拆成多句 |
| 被動語態 | 「被」字過多 | 改成主動句 |
| 公式化 | 首先...其次...最後 | 打亂順序或刪除標記 |
| 中式 AI 句型 | 「進行了一個...的」 | 直接用動詞表達 |
| 缺人味 | 事實羅列 | 加入經驗、自嘲、不確定 |
