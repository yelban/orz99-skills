# OpenCC 後校對：簡中→繁中轉換器的漏網清單

這份清單補強 `dictionary.md` —— 專門針對「**原文是簡體中文，已用 OpenCC `s2twp` 轉成繁體**」的文字做第二輪清洗。

## 為什麼需要這份清單

OpenCC `s2twp` 是機器字元替換，即使開了台灣用詞模式（`twp`），仍有三類問題它處理不了：

1. **詞彙層級的台灣用語**：OpenCC 只轉常見詞，較專業或較新的詞（如「聚類」「構建」「鏈路」）不在詞典裡，會原樣保留。
2. **一字多義的語境判斷**：例如 `connect` 在語料裡一律被當動詞轉成「連線」，但在「把它們 connect 到同一張圖」這種場景應該是「串連」；OpenCC 無法判斷。
3. **標點符號**：OpenCC 不碰標點，半形雙引號 `"..."`、半形逗號 `,`、半形單引號 `'...'` 會原封不動留下。

使用本清單時，逐項掃文章，抓出仍殘留的簡中用語或半形標點，改成台灣用法與全形標點。

## 一、OpenCC 漏轉的繁中用詞

### 技術／開發類

| OpenCC 後殘留 | 台灣用法 | 語境備註 |
|--------------|---------|---------|
| 聚類 | 分群 | 機器學習 clustering；「聚類」中國偏好，台灣兩者並用但正式寫作偏「分群」 |
| 社區（當 community 時） | 社群 | graphify/社交軟體/開源群組：community → 社群 |
| 構建 | 建構 / 建立 / 建好 | `build` 的動詞：建構（抽象）、建立（通用）、建好（完成態） |
| 註釋 | 註解 | 程式碼 comment 在台灣統一為「註解」 |
| 比如 | 例如 | 兩者都通，但正式寫作偏「例如」 |
| 後臺 | 背景 | 執行背景程序（background）：背景；管理後臺系統（backend admin）：後臺可保留 |
| PyPI 包 / 包 | PyPI 套件 / 套件 | package → 套件（Python、npm、Rust crate 皆然） |
| 命令（CLI） | 指令 | CLI command 在台灣正式寫作用「指令」；`install 命令`、`命令列`這類複合詞兩者並用 |
| 源位置 | 來源位置 | source location → 來源位置；source code → 原始碼（非「源代碼」） |
| 快取項 | 快取項目 | cache item/entry → 快取項目 |
| 暴露（非 API 術語） | 先提供 / 公開 | expose an API 在技術語境可保留「暴露」；泛用意義（「暴露問題」→ 呈現問題）應改 |
| 鏈路 | 流程 / 鏈 | auth chain → 認證流程；bootstrap chain → 啟動鏈 |
| 提供方 | 供應商 | provider → 供應商；service provider → 服務提供者或服務商 |
| 使用跟蹤 | 使用追蹤 | tracking → 追蹤（不是「跟蹤」） |
| 拉取 | 拉取 / 下載 | pull 在開源語境保留「拉取」無礙；日常語境偏「下載」 |
| 底層 | 底層 / 底下 | 技術語境保留，但「底層邏輯」屬互聯網黑話，見 dictionary.md |
| 項 | 項目 | item 單字 → 項目；但「條目」、「一項」這類複合用法不動 |
| 跨會話 | 跨對話 / 跨會話 | session → 對話（日常）或會話（技術）皆可，視讀者調整 |

### 動詞誤譯（OpenCC 轉字面但語境不對）

| 英文動詞 | OpenCC 易誤轉 | 應改為 | 判斷要點 |
|---------|-------------|--------|---------|
| connect (into a graph/set) | 連線 | 串連 | 「連線」= networking 術語；把節點加進圖/集合用「串連」 |
| link (as a verb) | 鏈接 / 連接 | 連結 | hyperlink → 連結；連接裝置 → 連接 |
| process | 處理 | 處理 / 進行 | 多數情境「處理」；「進行 X」常可改純動詞 |
| run | 運行 | 執行 | run code → 執行（「運行」是中國用語） |
| configure | 配置 | 設定 | config → 設定（「配置」在硬體分配語境可保留） |

### 錯字（OpenCC 或原文錯）

| 錯字 | 正字 | 說明 |
|------|------|------|
| 併（作「並」用時） | 並 | 「併」= 合併；「並」= 並且/並列。「併合併」屬錯字 |
| 象（作「像」用時） | 像 | image 中文一般用「像」；「對象」保留 |

## 二、標點符號

OpenCC 完全不處理標點。校對時務必替換：

| 半形 | 全形 | 語境 |
|------|------|------|
| `"..."` | 「...」 | 中文引文、強調詞；引用英文原文時保留半形 |
| `,` | ， | 中文句中；英文片段/程式碼保留半形 |
| `.` | 。 | 中文句尾；英文句尾/小數點保留半形 |
| `;` | ； | 中文句中分隔；程式碼保留半形 |
| `:` | ： | 中文；URL/時間/比值保留半形 |
| `!` `?` | ！？ | 中文；含英文字串的句子可保留半形 |
| `'...'` | 「...」或『...』 | 雙層引號用『』 |

**判斷原則**：中文字之間用全形，英文字之間或程式碼內用半形。跨語言邊界時，**以主要語言為準**：整句為中文就用全形，即使中間夾英文術語。

## 三、常見誤用校對檢查清單

校對 OpenCC 轉換後的文字時，搜尋以下字串快速抓違規：

```
# 殘留簡中/中國用語
rg -n '(聚類|社區[^$]|構建|註釋|比如|後臺|PyPI 包|源位置|快取項[^目]|暴露|鏈路|提供方|使用跟蹤|運行|配置(檔|文件))' file.md

# 錯字
rg -n '併(合併|發處理)' file.md   # 「併」當「並」用

# 半形標點（粗抓，會誤報英文片段）
rg -n '[\u4e00-\u9fff][",.;:!?][\u4e00-\u9fff]' file.md
```

grep 命中後仍需人工判斷：有些詞在特定語境（如程式碼註解、技術術語）保留是正確的。

## 四、使用範例

**輸入（OpenCC s2twp 剛轉完的繁中）**：
> 第二輪會並行呼叫 Claude 子代理處理文件、論文和圖片，用 Leiden 社群發現演算法做聚類，併把兩邊結果合併到一個 NetworkX 圖裡。每條關係都會被標記為 `EXTRACTED`。

**校對後**：
> 第二輪會並行呼叫 Claude 子代理，處理文件、論文和圖片，用 Leiden 社群發現演算法做分群，並把兩邊結果合併到 NetworkX 圖裡。每條關係都會標記為 `EXTRACTED`。

**修改點**：
1. `聚類` → `分群`（OpenCC 漏轉）
2. `併合併` → `並合併`（錯字，「併」應為「並」）
3. `都會被標記` → `都會標記`（被動改主動）
4. `合併到一個 NetworkX` → `合併到 NetworkX`（刪贅字「一個」）
5. `子代理處理` → `子代理，處理`（加逗號改氣口）
